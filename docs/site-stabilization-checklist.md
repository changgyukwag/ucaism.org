# Site Stabilization Checklist

Use this checklist before starting another major expansion phase.

## File and deployment health

- [ ] GitHub Desktop shows no unintended line-ending warnings.
- [ ] `.gitattributes` remains at repository root.
- [ ] Each new section has an `index.html` page.
- [ ] Each new JavaScript file is used by its corresponding page.
- [ ] Data JSON files are valid JSON.
- [ ] No broken internal links in newly added pages.

## Navigation

- [ ] Top navigation remains simple.
- [ ] Research pages are grouped logically.
- [ ] Reader-facing and institution-facing pages are not mixed confusingly.
- [ ] Mobile menu remains usable.

## Performance

- [ ] No unnecessary external scripts added.
- [ ] Images are compressed and sized appropriately.
- [ ] Page-level scripts use `defer` where possible.
- [ ] CSS additions are not duplicative.

## SEO and sharing

- [ ] `robots.txt` loads.
- [ ] `sitemap.xml` loads.
- [ ] Priority pages have unique titles and descriptions.
- [ ] Social sharing card image loads.
- [ ] JSON-LD is valid when inserted.

## Editorial credibility

- [ ] Claims are careful and institutionally appropriate.
- [ ] Placeholder pages are clearly marked as forthcoming or in development.
- [ ] UCAISM Global Initiative is described consistently.
- [ ] Intelligent Capitalism is described as forthcoming unless publication status changes.
